# ZInject 1.0 Release Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Ship ZInject 1.0 by rebranding from ZeroInject, adding `IServiceProviderIsKeyedService`, creating a package icon, and setting up CI/NuGet publishing with GitVersion + Release Please.

**Architecture:** Three workstreams executed sequentially — rebrand first (everything depends on new names), then feature work (`IServiceProviderIsKeyedService`), then CI/NuGet infrastructure. The rebrand is a bulk rename across ~56 files. The feature mirrors the existing `IServiceProviderIsService` pattern. CI uses GitHub Actions with GitVersion for versioning and Release Please for release automation.

**Tech Stack:** C# / .NET 8-10, Roslyn IIncrementalGenerator, xUnit, BenchmarkDotNet, GitHub Actions, GitVersion, Release Please, NuGet

**Step 3: Implement `EmitIsKnownKeyedService` in generator**

Add new method to `ZInjectGenerator.cs` after `EmitIsKnownService` (around line 2080):

```csharp
private static void EmitIsKnownKeyedService(
    StringBuilder sb,
    List<ServiceRegistrationInfo> keyedServices)
{
    sb.AppendLine("        protected override bool IsKnownKeyedService(global::System.Type serviceType, object serviceKey)");
    sb.AppendLine("        {");

    if (keyedServices.Count > 0)
    {
        sb.AppendLine("            if (serviceKey is string _key)");
        sb.AppendLine("            {");

        foreach (var svc in keyedServices)
        {
            var escapedKey = svc.Key!.Replace("\\", "\\\\").Replace("\"", "\\\"");
            foreach (var serviceType in GetServiceTypes(svc))
            {
                sb.AppendLine("                if (_key == \"" + escapedKey + "\" && serviceType == typeof(" + serviceType + ")) return true;");
            }
        }

        sb.AppendLine("            }");
    }

    sb.AppendLine("            return false;");
    sb.AppendLine("        }");
}
```

### Task 16: Create NuGet publish workflow

**Files:**
- Create: `.github/workflows/publish.yml`

**Step 1: Create `.github/workflows/publish.yml`**

```yaml
name: Publish to NuGet

on:
  release:
    types: [published]

permissions:
  contents: read

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Full history for GitVersion

      - name: Setup .NET
        uses: actions/setup-dotnet@v4
        with:
          dotnet-version: |
            8.0.x
            9.0.x
            10.0.x

      - name: Install GitVersion
        uses: gittools/actions/gitversion/setup@v3
        with:
          versionSpec: '6.x'

      - name: Determine version
        id: gitversion
        uses: gittools/actions/gitversion/execute@v3

      - name: Build
        run: dotnet build ZInject.slnx --configuration Release -p:Version=${{ steps.gitversion.outputs.semVer }}

      - name: Test
        run: dotnet test tests/ZInject.Tests/ZInject.Tests.csproj --configuration Release --no-build --verbosity normal

      - name: Pack
        run: dotnet pack ZInject.slnx --configuration Release --no-build --output ./artifacts -p:Version=${{ steps.gitversion.outputs.semVer }}

      - name: Push to NuGet
        run: dotnet nuget push ./artifacts/*.nupkg --source https://api.nuget.org/v3/index.json --api-key ${{ secrets.NUGET_API_KEY }} --skip-duplicate
```

**Step 2: Commit**

```bash
git add .github/workflows/publish.yml
git commit -m "ci: add NuGet publish workflow triggered by releases"
```
