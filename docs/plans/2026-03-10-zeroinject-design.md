# ZeroInject — Design Document

## Overview

ZeroInject is a compile-time DI registration library for .NET. It uses Roslyn source generators to auto-discover services via `[Transient]`, `[Scoped]`, `[Singleton]` attributes and generates `IServiceCollection` extension methods at compile time. No runtime reflection, no scanning.

## Phases

### Phase 1: Source-Generated Registration (Current)

Generate `IServiceCollection.AddXxxServices()` extension methods from attributes. Works with Microsoft's default DI container.

### Phase 2: Source-Generated DI Container (Future)

Replace the runtime resolution engine with a source-generated `IServiceProvider` implementation. All constructor calls baked in at compile time — zero reflection, zero `Activator.CreateInstance`. Plugs in via `services.BuildZeroInjectServiceProvider()` or `builder.UseZeroInjectContainer()`. Phase 1 attributes and discovery are shared; Phase 2 changes what the generator emits from registration calls to a full resolution engine.

### Attributes

```csharp
namespace ZeroInject;

[AttributeUsage(AttributeTargets.Class, AllowMultiple = false, Inherited = false)]
public abstract class ServiceAttribute : Attribute
{
    public Type? As { get; set; }           // Narrow registration to specific interface
    public string? Key { get; set; }        // Keyed service registration (.NET 8+)
    public bool AllowMultiple { get; set; } // false = TryAdd (default), true = Add
}

[AttributeUsage(AttributeTargets.Class, AllowMultiple = false, Inherited = false)]
public sealed class TransientAttribute : ServiceAttribute { }

[AttributeUsage(AttributeTargets.Class, AllowMultiple = false, Inherited = false)]
public sealed class ScopedAttribute : ServiceAttribute { }

[AttributeUsage(AttributeTargets.Class, AllowMultiple = false, Inherited = false)]
public sealed class SingletonAttribute : ServiceAttribute { }

// Assembly-level: override generated method name
[AttributeUsage(AttributeTargets.Assembly, AllowMultiple = false)]
public sealed class ZeroInjectAttribute : Attribute
{
    public ZeroInjectAttribute(string methodName) { }
}
```

### Registration Behavior

**Default:** Register against all implemented interfaces + concrete type, using `TryAdd` semantics.

**`As` property:** Narrows registration to a specific interface only.

**`Key` property:** Generates keyed service registration (`.NET 8+` only).

**`AllowMultiple = true`:** Switches from `TryAdd` to `Add` (allows duplicate registrations).

**Open generics:** Supported via `ServiceDescriptor` overload.

### Filtered Interfaces

These infrastructure interfaces are automatically excluded from registration:

- `System.IDisposable`
- `System.IAsyncDisposable`
- `System.IComparable`
- `System.IComparable<T>`
- `System.IEquatable<T>`
- `System.IFormattable`
- `System.ICloneable`
- `System.IConvertible`

### Generated Code Example

Input:
```csharp
[assembly: ZeroInject("AddDomainServices")]

[Transient]
public class OrderService : IOrderService, IDisposable { }

[Singleton(Key = "redis")]
public class RedisCache : ICache { }

[Scoped(As = typeof(IRepository<>))]
public class Repository<T> : IRepository<T>, IDisposable { }

[Transient(AllowMultiple = true)]
public class MyJob : IHostedService { }
```

Generated output (`ServiceCollectionExtensions.g.cs`):
```csharp
// <auto-generated/>
using Microsoft.Extensions.DependencyInjection;
using Microsoft.Extensions.DependencyInjection.Extensions;

namespace Microsoft.Extensions.DependencyInjection;

public static class MyDomainServiceCollectionExtensions
{
    public static IServiceCollection AddDomainServices(this IServiceCollection services)
    {
        // OrderService
        services.TryAddTransient<IOrderService, OrderService>();
        services.TryAddTransient<OrderService>();

        // RedisCache — keyed singleton
        services.TryAddKeyedSingleton<ICache, RedisCache>("redis");
        services.TryAddKeyedSingleton<RedisCache>("redis");

        // Repository<T> — open generic, narrowed via As
        services.TryAdd(ServiceDescriptor.Scoped(typeof(IRepository<>), typeof(Repository<>)));

        // MyJob — AllowMultiple
        services.AddTransient<IHostedService, MyJob>();
        services.AddTransient<MyJob>();

        return services;
    }
}
```

### Method Naming Convention

- Default: derived from assembly name (`MyApp.Domain` → `AddMyAppDomainServices()`)
- Override: `[assembly: ZeroInject("AddDomainServices")]`

### Generated Extension Method Namespace

`Microsoft.Extensions.DependencyInjection` — discoverable without extra usings.

### Scope

Per-assembly. Each project with the generator gets its own extension method. Consumer calls each explicitly.

### Diagnostics

| ID | Severity | Description |
|---|---|---|
| ZI001 | Error | Multiple lifetime attributes on same class |
| ZI002 | Error | Attribute on non-class type (struct, record struct) |
| ZI003 | Error | Attribute on abstract or static class |
| ZI004 | Error | `As` type not implemented by the class |
| ZI005 | Error | `Key` used on target below net8.0 |
| ZI006 | Warning | Class has no public constructor |
| ZI007 | Warning | Class implements no interfaces (registers concrete only) |
| ZI008 | Warning | `Microsoft.Extensions.DependencyInjection.Abstractions` not referenced |

### Generator Architecture

Single `IIncrementalGenerator` implementation:
1. Syntax provider finds all classes with `[Transient]`, `[Scoped]`, or `[Singleton]`
2. Extracts: type name, namespace, lifetime, interfaces, `As`, `Key`, `AllowMultiple`, open generic info
3. Collects all service infos
4. Emits single `ServiceCollectionExtensions.g.cs`

Internal data model implements `IEquatable<T>` for incremental generator caching/deduplication.
