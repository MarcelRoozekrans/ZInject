# Phase 3: Generated Container Replacement

## Goal

Replace MS DI's runtime resolution engine with a source-generated `IServiceProvider` implementation. ZeroInject-attributed services resolve via direct type-checks and `new` calls — zero dictionary lookups, zero reflection, zero call site chains. Framework services fall back to an inner MS DI provider.

## Architecture

The generator emits a complete `IServiceProvider` subclass with type-check based resolution. Runtime base classes in a new `ZeroInject.Container` package handle mechanical concerns (scoping, disposal tracking, thread-safe singleton initialization). The generated container is opt-in — existing Phase 2 factory lambda behavior remains the default when `ZeroInject.Container` is not referenced.

## Package Structure

### Existing (unchanged)
- **`ZeroInject`** — Attributes (`[Transient]`, `[Scoped]`, `[Singleton]`, etc.)
- **`ZeroInject.Generator`** — Roslyn source generator (extended to emit container code when `ZeroInject.Container` is referenced)

### New
- **`ZeroInject.Container`** — Runtime base classes (`net8.0;net10.0`). Contains `ZeroInjectServiceProviderBase`, `ZeroInjectScope`, and supporting infrastructure.

### Consumer dependency chain
```
App --> ZeroInject              (attributes)
    --> ZeroInject.Generator    (analyzer, source generator)
    --> ZeroInject.Container    (runtime base classes, opt-in)
```

The generator detects whether `ZeroInject.Container` is referenced in the compilation. If yes, it emits the generated container alongside the existing `IServiceCollection` extension method. If no, only the Phase 2 factory lambda extension method is emitted.

## Runtime Base Classes

### `ZeroInjectServiceProviderBase`

Abstract class implementing `IServiceProvider`, `IServiceScopeFactory`, `IDisposable`, `IAsyncDisposable`.

```csharp
public abstract class ZeroInjectServiceProviderBase : IServiceProvider, IServiceScopeFactory, IDisposable, IAsyncDisposable
{
    private readonly IServiceProvider _fallback;

    protected abstract object? ResolveKnown(Type serviceType);
    protected abstract ZeroInjectScope CreateScopeCore();

    public object? GetService(Type serviceType)
    {
        // 1. Try generated container first
        var result = ResolveKnown(serviceType);
        if (result != null) return result;

        // 2. Built-in services
        if (serviceType == typeof(IServiceProvider)) return this;
        if (serviceType == typeof(IServiceScopeFactory)) return this;

        // 3. Fallback to inner MS DI provider
        return _fallback.GetService(serviceType);
    }

    public IServiceScope CreateScope() => CreateScopeCore();
}
```

### `ZeroInjectScope`

Abstract class implementing `IServiceScope`, `IServiceProvider`, `IDisposable`, `IAsyncDisposable`.

```csharp
public abstract class ZeroInjectScope : IServiceScope, IServiceProvider, IDisposable, IAsyncDisposable
{
    private readonly ZeroInjectServiceProviderBase _root;
    private readonly IServiceScope _fallbackScope;
    private List<IDisposable>? _disposables;

    protected ZeroInjectServiceProviderBase Root => _root;

    protected abstract object? ResolveScopedKnown(Type serviceType);

    // Tracks IDisposable transients for disposal when scope ends
    protected T TrackDisposable<T>(T instance) where T : IDisposable;

    public object? GetService(Type serviceType)
    {
        var result = ResolveScopedKnown(serviceType);
        if (result != null) return result;

        if (serviceType == typeof(IServiceProvider)) return this;
        if (serviceType == typeof(IServiceScopeFactory)) return _root;

        return _fallbackScope.ServiceProvider.GetService(serviceType);
    }

    public void Dispose() { /* dispose tracked transients, scoped instances, fallback scope */ }
}
```

### Lifetime handling

| Lifetime | Storage | Thread safety | Location |
|----------|---------|---------------|----------|
| Singleton | Field on root provider | `Interlocked.CompareExchange` | `ResolveKnown` |
| Scoped | Field on scope | `??=` (scopes are single-threaded by convention) | `ResolveScopedKnown` |
| Transient | Created fresh each call | None needed | Both `ResolveKnown` and `ResolveScopedKnown` |

## Generated Code

For services:
```csharp
[Transient] public class GreetingService : IGreetingService { }
[Transient] public class OrderService : IOrderService {
    public OrderService(IGreetingService greeting) { }
}
[Singleton] public class Cache : ICache { }
[Scoped] public class ScopedWorker { }
```

The generator emits:

```csharp
// <auto-generated />
internal sealed class ZeroInjectAppServiceProvider : ZeroInjectServiceProviderBase
{
    private Cache? _singleton_Cache;

    public ZeroInjectAppServiceProvider(IServiceProvider fallback) : base(fallback) { }

    protected override object? ResolveKnown(Type serviceType)
    {
        if (serviceType == typeof(IGreetingService)) return new GreetingService();
        if (serviceType == typeof(IOrderService))
            return new OrderService((IGreetingService)GetService(typeof(IGreetingService))!);
        if (serviceType == typeof(ICache))
            return Interlocked.CompareExchange(ref _singleton_Cache, new Cache(), null) ?? _singleton_Cache;
        return null; // scoped types and unknowns fall through
    }

    protected override ZeroInjectScope CreateScopeCore()
        => new ZeroInjectAppScope(this, CreateFallbackScope());

    private sealed class ZeroInjectAppScope : ZeroInjectScope
    {
        private ScopedWorker? _scoped_ScopedWorker;

        public ZeroInjectAppScope(ZeroInjectAppServiceProvider root, IServiceScope fallbackScope)
            : base(root, fallbackScope) { }

        protected override object? ResolveScopedKnown(Type serviceType)
        {
            if (serviceType == typeof(IGreetingService)) return new GreetingService();
            if (serviceType == typeof(IOrderService))
                return new OrderService((IGreetingService)GetService(typeof(IGreetingService))!);
            if (serviceType == typeof(ICache)) return Root.GetService(serviceType);
            if (serviceType == typeof(ScopedWorker))
                return _scoped_ScopedWorker ??= new ScopedWorker();
            return null;
        }
    }
}
```

### Resolution strategy (shallow)

Dependencies are resolved via `(T)GetService(typeof(T))!` — a self-call on the container. This means:
- ZeroInject services depending on other ZeroInject services resolve through the type-check chain
- ZeroInject services depending on framework services (e.g., `ILogger<T>`) fall through to the inner MS DI provider naturally
- No dependency graph analysis needed in the generator
- Lifetime management (singleton caching, scoped caching) stays centralized in `GetService`

### Keyed services

Keyed services add `IKeyedServiceProvider` implementation with a separate `ResolveKeyed(Type, object?)` method using the same type-check pattern.

### Open generics

Open generics cannot be resolved at compile time. They return `null` from `ResolveKnown` and are handled by the fallback MS DI provider.

## Consumer API

### Console app
```csharp
var services = new ServiceCollection();
services.AddLogging();
var provider = services.BuildZeroInjectServiceProvider(); // generated extension method
```

### ASP.NET Core
```csharp
var builder = WebApplication.CreateBuilder(args);
builder.Host.UseServiceProviderFactory(new ZeroInjectServiceProviderFactory()); // generated class
```

Both `BuildZeroInjectServiceProvider()` and `ZeroInjectServiceProviderFactory` are generated (not in the runtime package) because they reference the generated `ZeroInjectAppServiceProvider` class directly.

### How `BuildZeroInjectServiceProvider()` works
1. Build an inner MS DI `ServiceProvider` from the `IServiceCollection` (for framework services)
2. Create the generated `ZeroInjectAppServiceProvider` with that inner provider as fallback
3. Return as `IServiceProvider`

## v1 Limitations

- **`IEnumerable<T>` resolution**: Delegates to fallback provider. ZeroInject services are not included in collection resolution. Future: generate collection resolution that merges ZeroInject + fallback services.
- **Open generics**: Delegate to fallback provider.
- **`IServiceProviderIsService`**: Delegates to fallback provider. ZeroInject services are not reported as available.

## Expected Performance

| Scenario | MS DI | ZeroInject Phase 2 (factory lambda) | ZeroInject Phase 3 (generated container) |
|----------|-------|--------------------------------------|------------------------------------------|
| Transient (no deps) | ~14 ns | ~15 ns | ~3-5 ns (type check + new) |
| Transient (1 dep) | ~16 ns | ~42 ns | ~6-10 ns (type check + new + self-call) |
| Transient (2 deps) | ~29 ns | ~92 ns | ~10-15 ns (type check + new + 2 self-calls) |
| Singleton | ~7 ns | ~8 ns | ~3-5 ns (field read) |
| Scoped | ~5.7 us | ~7.1 us | ~3-5 ns (field read) |

## Diagnostics

All existing diagnostics (ZI001-ZI010) remain unchanged. The generator performs the same constructor analysis regardless of whether it emits factory lambdas or container code.
